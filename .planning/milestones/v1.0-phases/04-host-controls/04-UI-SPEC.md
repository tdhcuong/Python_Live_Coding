## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot-to-text gaps in participant list, inline padding |
| sm | 8px | Compact element spacing (badge margins, button group gaps) |
| md | 16px | Default element padding (toolbar, problem panel body) |
| lg | 24px | Section padding (problem panel, sidebar) |
| xl | 32px | Layout gaps (header padding) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in room view) |

Exceptions:
- Timer display touch area: minimum 44px height (accessibility — touch target floor)
- Problem textarea: minimum 80px height (3 rows) so it does not feel cramped on expand

## Interaction States

### Run Button State Machine

| State | Label | Class |
|-------|-------|-------|
| Idle | "Run" | `bg-indigo-600 hover:bg-indigo-500` |
| Running (all clients) | "Running..." | `bg-indigo-600 opacity-50 cursor-not-allowed disabled` |
| Idle after result | "Run" | `bg-indigo-600 hover:bg-indigo-500` |

Transition trigger: `execution_start` → running; `execution_result` → idle.
Initial state on reconnect: determined by `room_state.is_running` field.

### Timer Color Progression

| Condition | Color Class | Animation |
|-----------|-------------|-----------|
| No timer / inactive | `text-gray-400` | none |
| Timer active > 300s | `text-gray-200` | none |
| Timer active ≤ 300s (5 min) | `text-amber-400` | none |
| Timer active ≤ 60s (1 min) | `text-red-400` | `animate-pulse` |
| Timer expired | `text-red-400` | none (pulse stops) |

Transition: CSS class swap on every `setInterval` tick (500ms).

### Timer Expiry Alert

Visual: entire timer display flashes via `@keyframes` — a 3-cycle flash animation on `#timer-display`, then settles to static red "00:00".

```css
@keyframes timer-flash {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.2; }
}
/* Applied on expire: animation: timer-flash 0.4s ease-in-out 3; */
```

Audio: Web Audio API beep — 880 Hz sine wave, 0.4s, gain 0.3 → ramp to 0.001.
- `AudioContext` created on "Start" button click (user gesture) to satisfy browser autoplay policy.
- On timer expiry (`setInterval` callback), call `audioCtx.resume().then(() => playBeep(...))`.
- Silent fallback: if `AudioContext` unavailable (unsupported browser), omit the beep — visual alert alone is sufficient.

### Problem Panel Expand/Collapse

- Default: collapsed (`#problem-body` has class `hidden`) when no problem is set.
- On `problem_update`: remove `hidden` from `#problem-body`, rotate chevron 180deg — panel auto-expands.
- On chevron click: `toggle('hidden')` on body, `toggle('rotate-180')` on chevron.
- Transition: `transition-transform duration-200` on chevron rotation only. Body shows/hides instantly (no height animation — keeps implementation simple).

### Host vs Participant View

| Element | Host sees | Participant sees |
|---------|-----------|-----------------|
| Problem panel header + body | Yes | Yes |
| Set Problem textarea + button | Yes | No |
| Timer display (digits) | Yes | Yes |
| Start Timer controls | Yes | No |
| Reset Editor button | Yes | No |

Detection: `hostToken` variable in room.js closure. If non-null, render host-only DOM fragments. If null, render participant-only view. No server-side visibility — purely client-side based on sessionStorage value.

## Layout Structure

### Target DOM Structure

```
div.flex.flex-col.h-screen                          ← outer, h-screen not min-h-screen
  header.shrink-0                                   ← header, unchanged + timer controls
  div#problem-panel.shrink-0                        ← conditional, collapsed by default
  div.flex.flex-1.overflow-hidden.min-h-0           ← flex row, min-h-0 critical
    aside.w-56.shrink-0                             ← sidebar (unchanged)
    div.flex.flex-col.flex-1.overflow-hidden.min-h-0  ← new editor+output wrapper
      main#editor-container.flex-1.overflow-hidden  ← CodeMirror, flex-1 (not height:100%)
      div#toolbar.shrink-0                          ← Run + Clear strip
      div#output-panel                              ← ~35% fixed height, overflow-y: auto
```

Critical CSS notes:
- Outer container: `h-screen` (not `min-h-screen`) — flex children need a fixed height anchor to distribute space correctly.
- Editor+output wrapper: `min-h-0` — without this, flex children cannot shrink below content height, causing overflow.
- `mainEl.style.height = '100%'` in `onRoomState` (room.js line 129/130) must become `mainEl.classList.add('flex-1')` — the style.height assignment must be removed.
- Output panel height: `height: 35vh` or `height: calc(35%)` — not flex-1, not auto. Fixed percentage of viewport.

**Source:** RESEARCH.md "Frontend: Layout Restructure" section; Pitfall 5 from RESEARCH.md (min-h-0 requirement).

## Animation and Transition Contracts

| Element | Animation | Duration | Trigger |
|---------|-----------|----------|---------|
| Chevron rotate | `rotate-180` CSS transform | 200ms ease | Problem panel toggle click |
| Toast appear/fade | `opacity-100` → `opacity-0` | 300ms | Auto after 3s (existing pattern) |
| Timer expiry flash | `@keyframes timer-flash` (3 cycles) | 0.4s × 3 = 1.2s total | `remaining === 0` |
| Timer pulse | `animate-pulse` (Tailwind built-in) | continuous | `remaining ≤ 60` |
| Button transitions | `transition-colors duration-150` | 150ms | hover/active |

No page transitions. No skeleton loaders. No scroll animations.

## Accessibility Notes

- All interactive buttons must have explicit `type="button"` to prevent accidental form submission.
- Timer display must use `aria-live="polite"` on `#timer-digits` so screen readers announce changes.
- Problem panel toggle button must have `aria-expanded` attribute reflecting collapsed/expanded state.
- Run button `disabled` attribute must be set (not just opacity) so keyboard users cannot trigger execution twice.
- `textContent` used exclusively for all user-supplied strings (problem text, output, names).
- Minimum touch target: 44px height on timer controls and preset buttons (use `min-h-[44px]` or `py-3` where needed).
